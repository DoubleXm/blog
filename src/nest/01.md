## 模块概述

在标准的开发流程中，会涉及到一些核心的概念：

- `dto` 全名 `Data Transfer Object`，数据传输对象，就是客户端往服务端传递的数据对象；

- `entity` 数据的实体对象，通常会和数据库中的表结构一一对应；也成为数据映射对象；

- `vo` 全名 `View Object`，视图对象，服务端返回给客户端的数据对象；

- `controller` 控制器，负责处理客户端的请求，接收参数，并决定调用哪个 `service` 来处理业务逻辑；并最终返回结果给客户端；

- `service` 服务，负责处理具体的业务逻辑；通常会调用 `repository` 来操作数据库；

- `module` 模块，将相关的 `controller` 和 `service` 组织在一起形成一个独立的功能模块；

- `di` 依赖注入，全名 `Dependency Injection`，是一种设计模式，通过将依赖关系注入到对象中来实现解耦；在 `nestjs` 中通过构造函数注入的方式来实现依赖注入；

## 初始化项目 - `app.controller.ts`

```ts :no-line-numbers
import { Controller, Get } from '@nestjs/common';
import { AppService } from './app.service';

@Controller()
export class AppController {
  constructor(private readonly appService: AppService) {}

  @Get()
  getHello(): string {
    return this.appService.getHello();
  }
}
```

在 `constructor` 中注入 `AppService`，随后在对应的路由方法中调用对应的逻辑；

## 初始化项目 - `app.service.ts`

```ts :no-line-numbers
import { Injectable } from '@nestjs/common';

@Injectable()
export class AppService {
  getHello(): string {
    return 'Hello World!';
  }
}
```

通过 `Injectable` 装饰器将 `AppService` 标记为一个可注入的服务；也可以称之为**提供者**

## 初始化项目 - `app.module.ts`

```ts :no-line-numbers
import { Module } from '@nestjs/common';
import { AppController } from './app.controller';
import { AppService } from './app.service';

@Module({
  imports: [],
  controllers: [AppController],
  providers: [AppService],
})
export class AppModule {}
```

在 `module` 中有四个比较重要的属性：

- `providers` 提供者，接收注入的服务集合；

- `controllers` 控制器，当前模块的控制器集合；

- `imports` 导入，当前模块依赖的其他模块；

- `exports` 导出，当前模块需要被其他模块使用的提供者（`service`）；

换句话说，`providers` 中基本存放的都是 `service`，

`imports` 中导入的则是你需要使用的其他模块；而 `exports` 则是你需要被其他模块使用的服务；比如购物车模块需要用户的信息，代码可能就是下面这种；

```ts :no-line-numbers
// cart.module.ts
import { Module } from '@nestjs/common';
import { CartService } from './cart.service';
import { UserModule } from '../user/user.module';

@Module({
  imports: [UserModule], // 导入 UserModule 模块
  providers: [CartService],
})
export class CartModule {}

// user.module.ts
import { Module } from '@nestjs/common';
import { UserService } from './user.service';

@Module({
  providers: [UserService],
  exports: [UserService], // 导出 UserService 服务
})
export class UserModule {}
```

## 初始化项目 - `main.ts`

```ts :no-line-numbers
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  await app.listen(process.env.PORT ?? 3000);
}
void bootstrap();
```

将 `AppModule` 作为根模块创建服务器实例，并监听对应的端口；
