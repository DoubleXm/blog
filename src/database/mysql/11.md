
## 视图

> [!INFO]
> 视图是一种虚拟存在的表， 它只保存 SQL 逻辑，不保存查询结果

```sql
-- 创建视图
-- 创建一个虚拟表字段为 id, name, age 数据时 emp id 大于等于 5 的数据
create view emp_v1 as select id, name, age from emp where id <= 5;

-- 查询视图
select * from emp_v1 where id <= 5;

-- 修改视图 使用 or replace 或者 alter
create view or replace emp_v1 as select id, name, age, salary from emp where id <= 5;
alter view emp_v1 as select id, name, age, salary from emp where id <= 5;

-- 删除视图, 也可以增加条件
drop view emp_v1;
drop view is exists emp_v1;
```

## with check option 限制

对视图数据的增删改都会落到基表中，就算是不满足 `where` 条件也会；

```sql
create view emp_v1 as select * from emp where id >= 10;

insert into emp_v1 values (9, 'zs'); -- 可以成功
insert into emp_v1 values (11, 'ls'); -- 也可以成功，但是视图中查不到这条数据
```

通过设置 `with check option` 让不满足 `where` 条件的语句失败；创建或者修改索引时使用；

```sql
create view emp_v1 as select * from emp where id >= 10 with check option;

alter view emp_v1 as select * from emp where id >= 10 with check option;
```

`with check opiont` 默认是级联级别完全写法是 `with cascaded check option` 级联检查的含义；

视图也支持**基于视图创建视图**， `cascaded` 就是所有的视图都要满足条件才能操作数据；

## 视图更新条件

视图中的行与基表中的行必须存在一对一的关系，如果存在以下情况，视图将不可更新；

- 聚合函数或者窗口函数
- 出现了 `distinct | group by | having | union`

## 视图的作用

- 简化操作： 对于多表联查的数据可以创建视图，后续直接查询视图就像单表查询一样，注意的是要给多表中相同的字段增加别名；
- 安全：对于表中的敏感字段可以通过视图过滤出来，让某些用户只查询不敏感的字段
