>[!INFO]
> 索引是一种有序的数据结构，它可以帮助 `MySQL` 高效获取数据；
>
> 但在增删改的场景下，索引节点也需要进行同步操作，会降低效率；
>
> 不同的存储引擎使用的索引也是不同的；

| 索引 | Innodb | MyIsam | Memory  |
| ------------- | :-----------: | :-----------: | :-----------: |
| B+Tree（常用） | ✅ | ✅ | ✅ |
| Hash |❌ | ❌ | ❌ | 
| Full-Text 全文索引| 5.6 之后支持 | ✅ | ❌ |
| R-Tree 地理空间索引|❌ | ✅ | ❌ |


## 索引分类

- 主键索引
- 唯一索引
- 常规索引
- 全文索引

根据存储形式不同

- 聚集索引
- 二级索引（辅助索引，非聚集索引）

如果存在主键，逐渐索引就是聚集索引；

如果不存在逐渐，第一个唯一字段，将会被作为聚集索引；

如果没有主见，也没有唯一字段，`innodb` 会自动生成一个 `rowid`，作为隐藏的聚集索引

## 索引语法

```sql
-- unique 唯一索引
-- fulltext 全文索引
-- 不写表示默认索引
create {unique | fulltext} index 索引名称 on 表名(字段名...);

-- 查看索引
show index from 表名;

-- 删除索引
drop index 索引名称 on 表名;
```
