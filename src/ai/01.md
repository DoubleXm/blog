## 通识

从技术视角看，大模型本质上是一个基于参数的概率模型。它不会逐字保存训练资料，而是把训练数据中的语言结构、知识关联和推理模式压缩到参数中；推理时再根据当前上下文，计算下一个 token 的概率分布。

大语言模型通常写作 `LLM`，全称是 `Large Language Model`。它是一类以自然语言和代码为主要建模对象的深度学习模型。

大语言模型能力主要来自三件事：

- 算法：主流 LLM 基于 `Transformer`，核心机制是 `attention`。token 会先映射成向量，再经过多层权重矩阵和 attention 计算，动态建立上下文中不同 token 之间的依赖关系。
- 数据：训练语料决定模型能学习到哪些语言模式、事实关联、代码结构和表达风格。数据质量、覆盖范围和清洗策略，通常比单纯堆数据量更关键。
- 算力：训练阶段依赖大量 GPU/加速卡反复更新参数；推理阶段则不断基于上下文计算下一个 token。参数越多、上下文越长，通常意味着更高的显存、延迟和调用成本。

核心概念：

- 参数：模型内部可学习的权重数字。`10M ≈ 1,000 万`，`1B ≈ 10 亿`，`10B ≈ 100 亿`，`1T ≈ 1 万亿`。参数多通常代表容量更大，但不等于事实一定更新、推理一定更强。
- token：模型读写文本的最小计费/计算单位，可能是一个字、一个词，也可能是半个词。粗略看，中文 `1 字 ≈ 1 token`，英文 `1 个词 ≈ 1~2 token`。
- token 规模：`1K ≈ 1,000 token`，`10K ≈ 1 万 token`，`1M ≈ 100 万 token`。输入、输出、工具结果、历史对话都会占 token；输出越长，消耗越多。
- 向量：多维数字坐标，用来表示 token 的含义和特征。比如“北京”和“上海”的坐标会更接近，“北京”和“香蕉”会更远；如果做搜索，会用坐标距离找相近内容，但大模型生成回答时，主要是把这些向量继续交给参数权重和 attention 计算。
- 权重：模型训练后留下来的大量数字，可以理解成被压缩进模型里的经验规律。比如看到“北京是中国的”，权重会让“首都”比“水果”更可能出现。
- attention：注意力机制。在计算当前 token 时，判断上下文里哪些 token 更值得参考。比如“苹果发布了新手机”，模型会更关注“发布”“新手机”，而不是把“苹果”优先理解成水果。
- 上下文：这次请求里模型能看到的全部内容。比如一次会话里的系统提示、前面所有问答、当前问题、工具返回结果、文件片段，以及应用主动塞回来的思考摘要或执行记录，都可以成为上下文；没放进上下文的内容，模型这次就看不到，所以模型默认是没有记忆的。
- 上下文窗口：模型一次能容纳的上下文大小。超过窗口后，应用通常只能截断、摘要或丢掉一部分旧内容；被丢掉的内容模型无法继续参考，所以会表现得像“忘了前文、变笨了”。
- 缓存命中：多次请求前缀相同，服务商可能复用已计算的上下文。命中时通常更快、更便宜；前面内容一变，缓存就容易失效。
- Agent：基于 LLM 组织出来的执行系统。它会让模型在循环中理解目标、拆分步骤、调用工具或 API、观察结果，再继续决定下一步；可以理解为 `LLM + 工具 + 状态/记忆 + 任务规划`。如果说 LLM  是大脑，那么 Agent 就是手和脚。

一次对话的流程如下：

1. 应用把系统提示、历史对话、用户输入和工具说明拼成上下文。
2. 模型把文本切成 token，再把 token 转成内部向量。
3. `Transformer` 用参数权重和 attention 计算 token 之间的关系。
4. 模型得到“下一个 token”的概率分布，选出一个 token。
5. 把选出的 token 拼回上下文，重复第 3-5 步，直到回答结束、触发工具调用或达到长度限制。

![大模型逐个 token 生成回复流程](/ai/model-token-generation.svg)

常见生成参数会影响“怎么选下一个 token”：

- `temperature`：温度越低，越偏向选择高概率 token，回答更稳定、更保守；温度越高，低概率候选也更容易被选中，回答更多样，但也更容易跑偏。
- `top_p`：只在累计概率达到指定比例的候选 token 里抽样，常用于控制随机范围。
- `top_k`：只在概率最高的前 `K` 个候选 token 里抽样，`K` 越小越保守。
- `max_tokens`：限制最多生成多少 token，决定回复长度上限。
- `frequency_penalty`：降低已经频繁出现过的 token 概率，减少重复表达。
- `presence_penalty`：只要某类内容出现过，就降低它继续出现的概率，鼓励模型引入新内容。

## 初次调用 

> 在大模型的调用过程中，可以直接使用接口进行调用； 也可以使用 sdk 进行调用； 大多数不会使用这两种，`deepseek | openai` 为例
>
> - 由于各个厂商都有自己的 api 参数上虽然大同小异，通过该方案开发出来的智能体仍然存在迁移困难
> - 使用 sdk 也存在类似的问题，虽然没有各种 `coding plain` 风格的 api，但是 `openai | anthropic` 也是不同风格，可以理解它就是 api 的特定语言翻译

```py
from requests import post
from app.core.config import settings

res = post(
    "https://api.deepseek.com/chat/completions",
    json={
        # 模型名称
        "model": "deepseek-v4-pro",
        "messages": [
            # 系统消息
            {"role": "system", "content": "你是日漫中七龙珠主角，孙悟空"},
            # 用户消息
            {"role": "user", "content": "你好，我是卡卡罗特"},
        ],
        # 流式输出
        "strame": "false",
    },
    headers={
        "Content-Type": "application/json",
        # 模型的 api key
        "Authorization": f"Bearer {settings.DEEPSEEK_API_KEY}",
    },
)

print(res.json())
```

使用 `sdk` 要预先下载 `uv add openai`

```py
from openai import OpenAI
from app.core.config import settings

client = OpenAI(
    api_key=settings.DEEPSEEK_API_KEY,

    messages=[

        {"role": "user", "content": "你好"},
    ],

print(response.model_dump_json())
```

## LangChain

是一个用来开发智能体工程（Agent Engineering）的框架，同时兼容 `py | ts` 两种语言；支持各种模型的调用；内部包含四个核心模块

- LangChain: 用于快速构建智能体，可兼容任何模型提供商
- LangGraph: 从底层一步步控制智能体的构建，包括记忆（Memory）、人机协同（HITL）等
- DeepAgents: 用于构建复杂的、处理多步骤的任务的智能体
- LangSmith: 用于测试、观察、评估、部署智能体

### 模型和 agent 创建

默认情况下 langchain 会自动读取环境变量中的 api key， 所以通过 `create_agent` 不需要指定 key， 但是变量名要符合官网规范 `OPENAI_API_KEY`


```py
# uv add langchain
from langchain.agents import create_agent

agent = create_agent(
    model="openai:gpt-5.5",
    system_prompt="你是一名天气助手",
)
result = agent.invoke(
    {
        "messages": [
            {"role": "user", "content": "今天天气怎么样"},
        ]
    }
)
print(result["messages"][-1].content_blocks)
```

但此种方式仅支持一些比较出名的厂商，像阿里云百炼等并不会提供，需要使用 `init_chat_model` 去创建一个 model，但是需要安装 `langchain-deepseek`

支持的所有外部模型列表 [model list](https://docs.langchain.com/oss/python/integrations/providers/overview)

创建出来的 model 可以直接丢给 `create_agent` 去使用

```py
import os
from langchain.chat_models import init_chat_model
from app.core.config import settings

DEEPSEEK_API_KEY = os.environ["DEEPSEEK_API_KEY"] = settings.DEEPSEEK_API_KEY

model = init_chat_model(
    model="deepseek-v4-flash",
    # 通过后续的选项转发到 openai
    model_provider="openai",
    api_key=settings.DEEPSEEK_API_KEY,
    base_url="https://api.deepseek.com",
)
# 阻塞
result = model.invoke("你是谁")
# 流
result = model.strame("你是谁")
```

`init_chat_model` 支持的关于模型常用参数包括

- max_retries 最大重试次数，默认 6
- timeout 超时等待时间 s
- max_tokens 最大响应 token 数
- temperature 控制模型随机性，越高越具有创造性，越低越确定性，取值范围 0 - 2 默认 1
- top_p 模型会考虑前 top_p 概率的 token 的结果。所以 0.1 就意味着只有包括在最高 10% 概率中的 token 会被考虑。默认是 1

完整调用

```py
import os
from langchain.agents import create_agent
from langchain.chat_models import init_chat_model
from app.core.config import settings

DEEPSEEK_API_KEY = os.environ["DEEPSEEK_API_KEY"] = settings.DEEPSEEK_API_KEY

model = init_chat_model(
    model="deepseek-v4-flash",
    model_provider="openai",
    api_key=settings.DEEPSEEK_API_KEY,
    base_url="https://api.deepseek.com",
    temperature=0.8,
    max_tokens=4000,
    top_p=1,
    timeout=20,
    max_retries=6,
)

agent = create_agent(
    model,
    system_prompt="你是七龙珠动漫中的克林",
)
results = agent.stream({"messages": [{"role": "user", "content": "我是 18 号"}]}, stream_mode="messages")

for chunk, _ in results:
    if chunk.content:
        print(chunk.content, end="", flush=True)
```

### Messages 消息

上面的消息一直都是在构造对象传递，实际上 `langchain` 中提供了很多种消息的对象，可以用来构造；

消息中主要包含三块内容

role： 角色主要包含 system（系统） user（用户） assistant（ai）
content： 消息的实际内容
metadata：消息的元数据

```py
from langchain.messages import HumanMessage, SystemMessage, AIMessage

#...

results = agent.invoke({
  "messages": [
    SystemMessage("你是七龙珠动漫中的克林"),
    HumanMessage("我是 18 号")
  ], 
})

for message in results["message"]:
    message.pretty_print() # 更好看的打印

"""
================================ System Message ================================

你是七龙珠动漫中的克林
================================ Human Message =================================

我是 18 号
================================== Ai Message ==================================

（微微一笑）18号！没想到会在这里遇见你。自从你和克林结婚后，我们也好久没见了。
"""
```

## 工具

是 `create_agents` 的一个参数，在最开始的时候，agent 会将上下文和工具包装好交给模型，然后模型分析需要使用哪个工具，在让 agent 执行工具，最后模型分析好在返回给用户

```py
from langchain.agents import create_agent
from langchain.chat_models import init_chat_model
from langchain.messages import HumanMessage
from langchain.tools import tool

from app.core.config import settings


@tool
def get_weather(location: str) -> None:
    """天气查询工具"""
    return f"{location} 今天是晴天"


model = init_chat_model(
    model="deepseek-v4-flash",
    model_provider="openai",
    base_url="https://api.deepseek.com",
    api_key=settings.DEEPSEEK_API_KEY,
)
agent = create_agent(model, tools=[get_weather])

result = agent.invoke({"messages": [HumanMessage("上海今天的天气怎么样")]})

for message in result["messages"]:
    print(message.pretty_print())

"""
================================ Human Message =================================

上海今天的天气怎么样
None
================================== Ai Message ==================================
Tool Calls:
  get_weather (call_00_9LXySUHiZlOC4Vq4r5EY8414)
 Call ID: call_00_9LXySUHiZlOC4Vq4r5EY8414
  Args:
    location: 上海
None
================================= Tool Message =================================
Name: get_weather

上海 今天是晴天
None
================================== Ai Message ==================================

上海今天**天气晴朗**，是个大晴天！☀️

如果您需要更详细的天气信息（如温度、风力等），请随时告诉我，我可以进一步帮您查询。
None
"""
```

工具的定义默认会使用工具函数的 doc 注释、参数、返回值作为 ai 识别的依据， 也可以直接给 `@tool` 传入参数

```py
@tool("获取天气", description="根据传入的地区获取天气")
def get_weather(location: str) -> None:
    return f"{location} 今天是晴天"
```
