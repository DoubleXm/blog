Deep Dive: Hand-rolled Watermark Logic

calendar_today 2024.05.20
timer 12 min read
terminal TypeScript / Canvas API

Creating a robust watermark component in React/Vue isn't just about overlaying a div. It requires careful consideration of security (preventing removal via DevTools), performance (avoiding re-renders), and visual subtlety. 在现代前端开发中，水印组件的安全性和性能是必须要考虑的关键指标。

MutationObserver Implementation TypeScript 
const observeWatermark = (target: HTMLElement) => {
  const observer = new MutationObserver((mutations) => {
    for (const mutation of mutations) {
      if (mutation.type === 'childList' || mutation.type === 'attributes') {
        // Anti-tamper logic / 防篡改逻辑
        recreateWatermark(target);
      }
    }
  });
  
  observer.observe(target, { 
    attributes: true, 
    childList: true, 
    subtree: true 
  });
};

The Canvas Strategy / 画布渲染策略

We leverage the Canvas API to generate a base64 pattern that is applied as a background image. This method is more memory-efficient than creating thousands of DOM elements for the watermark grid. 使用 Canvas API 生成 base64 图案并作为背景图片应用，这种方式比创建数千个 DOM 元素更为节省内存。

Performance Tip

By using memo() and useMemo, we ensure the canvas only regenerates when configuration props change.

info

Context Info

The MutationObserver detects if a user deletes the node or hides it via CSS visibility/display styles.

Security Constraints / 安全约束

Instead of hardcoding colors, we map the watermark opacity and color to the application's CSS variables, ensuring theme consistency.

warning
Caution Required

Over-securing watermarks can lead to performance degradation on low-end devices if the observer frequency is too high. 频繁的 DOM 观察可能会在低端设备上导致性能卡顿。

dangerous
Critical Failure Case

If the MutationObserver is disconnected or disabled via browser extensions, the anti-tamper logic will fail completely. 如果观察器被插件禁用，防篡改机制将失效。

Conclusion / 总结

A production-ready watermark component requires balancing security, performance, and user experience. 生产级的水印组件需要在安全性、性能和用户体验之间取得平衡。
